# Changelog

Notable changes to DocQFlow are recorded here so reviewers, teammates, and AI agents can quickly see what was added or changed and when.

## 2026-05-11

### Added

- `frontend/src/components/review/FieldsPanel.test.tsx`: new Vitest suite covering the rewritten FieldsPanel. 8 cases — every curated row renders, the MISSING badge appears for fields without a value (12 of 15 in the test fixture), the verdict strip renders when `pipelineResult` is non-null and hides when null, an issue with `field: '1 BLOCK & LOT'` resolves to the `parcel_number` row via `ACROFORM_TO_CANONICAL` and renders its message inline, unmatched issues (e.g. `'Check Box8'`) bucket into the "Other Findings" section, click-to-edit + Enter commits + the "Edited" badge appears, the Save button stays disabled until there are pending edits and toasts on click, Escape cancels without committing. Mocks `sonner` so the toast call is observable without rendering a real toast.

### Changed

- `frontend/src/components/review/FieldsPanel.tsx`: rewritten from a 9-field static list with a hardcoded "Auto-classification" footer into a 15-field editable surface that subsumes the old AssessmentPanel. New header strip surfaces the `PipelineResult` verdict pill (`pill-success|pill-warn|pill-danger` + `data-verdict` for tests), `llm_profile`, `latency_ms`, and total issue count when `pipelineResult` is provided; omitted entirely when null so the placeholder fallback path doesn't render a stale strip. The row list mirrors `src/api/pdf_fields.py` `_FIELD_MAP` — 15 canonical Form 3-8 fields covering application number, date filed, project address, parcel, cost, construction type, occupancy class, proposed use, stories, dwelling units, owner/lessee, contractor, contractor address, license, description. Each row is click-to-edit: clicking the value (or the pencil icon button) swaps in an `<input>` whose Enter commits to local `edits` state, Escape cancels, and `onBlur` commits. An "Edited" pill renders next to the label for any row with a pending edit. The Save button at the top stays disabled until `edits` is non-empty and on click calls `toast.success('Saved N field(s) locally (backend persistence pending)')` — backend persistence is intentionally deferred (no PATCH endpoint yet). Pipeline issues are bucketed by `groupIssuesByRow`: issues whose `Issue.field` resolves to a curated canonical key via `ACROFORM_TO_CANONICAL` attach inline below their row (severity pill + source pill with `SparkleIcon`/`ShieldIcon` + message + optional confidence percent), while orphan issues fall through to an "Other Findings" section at the bottom that renders the raw AcroForm field name as the row label. The pencil-icon button keeps `aria-label={`Edit ${label}`}` as the canonical edit affordance for screen readers; the value-display button has no aria-label so its visible text (or the "Missing" pill) is the accessible name, which also keeps `getByRole('button', { name: 'Edit Application Number' })` unambiguous for the test suite.
- `frontend/src/lib/pipelineFields.ts`: expanded the canonical key map from 7 entries to 15 to match the FieldsPanel rewrite, adding `application_number`, `date_filed`, `stories`, `dwelling_units`, `proposed_use`, `occupancy_class`, `construction_type`, `contractor_address`, `owner_name`, and `description`. Source field lookup is now multi-candidate (first non-null wins) so Form 3 widget names (e.g. `'5 NO OF STORIES OF OCCUPANCY'`) and Form 8 widget names (e.g. `'5A NO OF STORIES OF OCCUPANCY'`) share a canonical key without breaking either flow. Description is reconstructed by joining all populated `'16 DESCRIPTION'` / `'16A DESCRIPTION'` / `'16B'` / `'16C'` / `'16D'` widgets (mirrors the description-concat behavior in `src/api/pdf_fields.py`). New named export `ACROFORM_TO_CANONICAL`: the inverse map (verbatim AcroForm widget name → canonical key) used by FieldsPanel to attach a pipeline `Issue` to the row whose `field` it references. When multiple canonical keys share a source widget (e.g. both `address` and `project_address` point at `'1 STREET ADDRESS OF JOB BLOCK  LOT'`), the first-declared canonical wins so the inverse is deterministic. The `SOURCE_FIELDS` type widened from `as const satisfies Partial<Record<FieldName, string>>` to `Record<string, readonly string[]>` because the keys are no longer constrained to the legacy `FieldName` union (we added keys like `owner_name` and `description` that the legacy union didn't cover).
- `frontend/src/lib/pipelineFields.test.ts`: extended to cover every new canonical key on a realistic Form 3-8 fixture, plus a Form-8-only fixture asserting fallback to the `5A` / `9A` / `7A` / `8A` / `4A` widgets when the Form-3 variants are null. New `concatenates description widgets` test pins the join order. New `ACROFORM_TO_CANONICAL` describe block asserts the inverse map round-trips (`'1 BLOCK & LOT'` → `parcel_number`, `'14C CSLB'` → `license_number`, etc.), every description-widget variant maps to `description`, Form 8 fallbacks (`5A`, `9A`, `4A`) point to the same canonical as Form 3, and unhandled widgets (`'Check Box8'`, `ISSUED`) return `undefined`.
- `frontend/src/pages/Review.tsx`: removed the "Pipeline assessment" tab (and its `SparkleIcon` rail button), the `AssessmentPanel` import, and the derived-default-tab logic that promoted `'assessment'` whenever `pipelineResult` was non-null. The `Tab` type is now `'fields' | 'timeline' | 'history'` and `tab` is back to a plain `useState<Tab>('fields')`. `FieldsPanel` now receives `pipelineResult` directly and renders the verdict/issues there; the assessment surface and the fields surface are no longer split. Drops the `permit` prop from the FieldsPanel call site (FieldsPanel no longer renders the hardcoded "Auto-classification" footer that used `permit.department` / `permit.confidence`).
- `frontend/vite.config.ts`: proxy target now reads `VITE_API_URL` via `loadEnv(mode, process.cwd(), '')` and falls back to `'http://localhost:8000'` when unset, so local dev can point at the deployed Cloud Run backend (`https://docqflow-api-dev-a54smp66fa-uc.a.run.app`) via `frontend/.env.local` without editing the checked-in config. `frontend/.env.local` is already gitignored by `frontend/.gitignore:13` (the `*.local` pattern).

### Removed

- `frontend/src/components/review/AssessmentPanel.tsx` and `frontend/src/components/review/AssessmentPanel.test.tsx`: deleted. AssessmentPanel's verdict strip + issue list now live inside FieldsPanel; the component had no remaining consumers. The 7 vitest cases that used to cover AssessmentPanel are functionally replaced by the new `FieldsPanel.test.tsx` cases for verdict strip rendering, issue-to-row attachment, and "Other Findings".

## 2026-05-10

### Added

- `.github/workflows/test.yml` (docqflow-1po): new `deploy` job replaces the previously commented-out `docker-push` stub and removes the last manual step from the dev rollout. Gated on `github.ref == 'refs/heads/main' && github.event_name == 'push'` and `needs: [lint, test]`, so PRs never deploy and a failing lint/test blocks rollout. `timeout-minutes: 20` caps runaway builds; `concurrency: group=deploy-..., cancel-in-progress: false` serializes deploys from main so two pushes can't race on the traffic-promote step. Auths to GCP via Workload Identity Federation (`secrets.WIF_PROVIDER` → `projects/825166191827/locations/global/workloadIdentityPools/github-pool-dev/providers/github-provider-dev`, `secrets.WIF_SERVICE_ACCOUNT` → `docqflow-api-dev@docqflow.iam.gserviceaccount.com`) — no long-lived JSON keys. Build step uses `docker/build-push-action@v6` with `cache-from/to: type=gha,mode=max` and pushes BOTH `:latest` and `:${{ github.sha }}` to `us-central1-docker.pkg.dev/docqflow/docqflow/docqflow`. Deploy uses a **canary flow**: (1) `gcloud run services update docqflow-api-dev --image=...:${{ github.sha }} --no-traffic --tag=preflight` creates a new revision held at 0% traffic with a stable `preflight` tag; existing env vars / secrets / scaling / concurrency are preserved (no `--set-env-vars` or `--set-secrets` in CI). (2) The smoke test resolves the tag-specific URL via `gcloud run services describe ... | jq '.status.traffic[] | select(.tag=="preflight").url'` and curls `/health` in a 30×2s loop (~60s budget), failing the job on non-200. Because traffic is still on the prior revision while we probe, a failed smoke test is genuinely non-disruptive — the bad revision stays at 0% and no real user ever hits it. (3) Only if the smoke test passes does `gcloud run services update-traffic --to-latest` promote the new revision to 100%. **Rollback runbook:** `gcloud run services update-traffic docqflow-api-dev --region=us-central1 --to-revisions=<prior-revision>=100` (or redeploy: `gcloud run services update docqflow-api-dev --image=us-central1-docker.pkg.dev/docqflow/docqflow/docqflow:<prior-sha>`). SHA-pinned tags make rollback a copy-paste against `git log`. CodeRabbit caught the original draft of this job that omitted `--no-traffic` and would have immediately routed traffic to a broken revision; the canary flow is the corrected design.
- GCP IAM bootstrap (docqflow-1po, applied manually as a one-time setup, no IaC yet): `docqflow-api-dev@docqflow.iam.gserviceaccount.com` granted `roles/artifactregistry.writer` and `roles/run.admin` at the project level (lets CI push images to Artifact Registry and update the Cloud Run service), plus `roles/iam.serviceAccountUser` on itself (required so `gcloud run services update` can re-attach the SA as the runtime SA on the new revision). The pre-existing WIF setup — pool `github-pool-dev`, provider `github-provider-dev` with `attributeCondition: assertion.repository=='tomtranjr/docqflow'`, and `roles/iam.workloadIdentityUser` for `principalSet://.../attribute.repository/tomtranjr/docqflow` on the SA — is unchanged. GitHub repo secrets `WIF_PROVIDER` and `WIF_SERVICE_ACCOUNT` were added to back the new workflow. Note: the same SA is used for both CI deploy and Cloud Run runtime — acceptable for the dev/school env but a split (`docqflow-deployer` vs `docqflow-api-dev`) is the natural next step if a prod env ever lands.

### Changed

- `src/api/auth.py`, `src/api/config.py` (docqflow-h39, **temporary**): added a `DOCQFLOW_DISABLE_AUTH=true` env-var-gated bypass on `get_current_user_id`. When set, the dependency returns the fixed dev UUID `00000000-0000-0000-0000-00000000d0c0` without inspecting the `Authorization` header — POST attribution and GET RLS lookups both resolve to this user, so the Review page round-trips work. Backstory: PR #46 (commit 784da90) added Supabase JWT auth to `/api/documents/process` and `/api/documents/{sha256}`, but the frontend has no Supabase JS client wired in (no `signIn`, no session, no `Authorization` header on `processPDF` / `getDocument` calls). The previous prod was the pre-PR-46 manual deploy (commit e32618a), so the gap was hidden. Today's automated deploy (commit c16b598) promoted both halves of PR #45 (real-data wiring) + PR #46 (auth gate) into prod at once, and the pipeline endpoints started returning 401 to the anonymous frontend — which fell back to `usePlaceholderExtraction`, producing the visible regression where the Review page showed `Maria Hernandez` / `88 Sunset Blvd` / `Doe Construction` for every upload. This change unblocks the demo while docqflow-h39 builds out the magic-link login UI. To re-enable the gate after the frontend ships, unset `DOCQFLOW_DISABLE_AUTH` on the Cloud Run service (or delete the env var binding in `gcloud run services update`). New test `tests/pipeline/test_routes_pipeline.py::test_get_current_user_id_bypassed_when_disable_auth_flag_set` asserts the bypass returns `_DEV_USER_ID` only when the flag is on; the existing `test_process_requires_auth` continues to assert 401 when the flag is off.

- `frontend/src/pages/Review.tsx`: the previous document's pipeline assessment no longer leaks across `liveEntry` changes. Result: when `liveEntry` switches to a doc whose `pdf_sha256` is missing or whose `getDocument` fetch fails/404s, consumers (`tab` derivation, `fields` derivation, `<AssessmentPanel result={…}>`) see `null` until the new sha's fetch resolves — so the derived default tab (`manualTab ?? Surfaced by CodeRabbit on PR #47.
